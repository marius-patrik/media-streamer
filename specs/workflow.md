# Development Workflow

1. Select a spec and acceptance criteria.
2. Write an implementation packet from `specs/packet-template.md`.
3. Confirm new dependencies before editing manifests.
4. Implement only the packet scope.
5. Add tests matching acceptance criteria names.
6. Run verification commands.
7. Update specs when behavior changes.

## Packet Exit Criteria

- Spec sections are linked.
- Dependency changes are approved or absent.
- Tests and checks ran, or failures are documented with the blocking reason.
- No unrelated files changed.
